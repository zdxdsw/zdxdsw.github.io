What is this HIT about? +-

We are trying to build a question-answering system which is able to retrieve one or more images to answer a user's query.

To do this, in the first place we are going to investigate what type of query can potentially be answered by images.

For example, the following quries can be answered by one or more images given below:

  • Do MIT use the same animal on their "MIT Engineers" logo as their mascot?
  • Do MIT and Caltech use the same animal as their mascot?
  • Which other schools use the same animal as their mascot as Caltech?

Unfortunately, current question-answering systems largely depend on textual search and seldom try to find an answer supported by visual information. For example, search engines like Google and Bing are looking for exact match between the input query and large textual corpora. However, some queries do not necessarily have answers lying in text. And we are interested in such queries because we believe that the next generation of search engines should be able to find answers to user queries from visual data. Therefore, this HIT is designed to collect queries whose answers happen to lie in some image(s). We appreciate your help!

Format: Each HIT starts with a group of 4 images from which we would like to get 3 queries such that answering the quries need to utilize information provided by one or more of those images.

A valid annotation should include:

  1. Select one or more images from the given group.
  2. Ask a question about the image(s) you select.
  3. Write down the correction answer of your query. The answer should lie in the image(s) you select.
Required properties of the query:
  • More easy to be answered by image(s) than by text: Since we are interested in queries that are NOT well handled by current search engines, if a query can be easily answered by Google/Bing/Wikipedia, it is not a good one! Some bad examples are provided below. If you type those queries in Google or Bing, you may find that the search engine already highlighted in answer on top of the result page.
    • Which street is Times Square located?
    • Who is the champion of 2018 world cup?
  • No referential phrases directly referring to image regions: Importantly, it is only for the convenience of this annotation task that we provide images before you come up with queries. However, in the testing setting of the question-answering system we are going to build, queries come before images. This means we would like to have systems figure out which image contains the answer after it sees the query. Therefore, please avoid phrases in the question that assume knowledge of which image to use, for example, "in this image", "this place", "the object on the right", "the person in blue shoes" etc. An easy way to check whether a question satisfies this requirement is to imagine someone asks you the question without providing any images, then think about whether you are able to fully understand what it is asking. In the following bad examples, we highlight the unclear references that should be avoided:
    • The statue at the center is located at which park?
    • Is there a bus station on the street?
  • Has definitive answer: The answer to your query cannot be too subjective or change from time to time.
    • How good is the view of Chicage from Willis Tower skydeck?  (Too subjective)
    • What is the weather in Manhattan?  (Time sensitive)
  • Not about simple facts: The query cannot be answered blindly based on common sense (i.e. without using visual contents)
    • Can a boat float on the water?
    • How many wheels does a car have?
  • Associated with multiple image regions: Answering the question requires attention to multiple image regions
  • Associated with more than one image: Information presented by different images complements each other to infer the answer. (This is only required in the third annotation, but also recommended in the first two)
Not required but highly recommended properties of the query:
  • Non-binary: A binary question has only yes/no answer. We prefer questions that start with "what", "where", "which", etc., to binary questions.
We highly recommend that you check the examples for a better sense what is a good response.

Hints for asking a "multi-image" question +-

We understand that the "multi-image" requirement for Annotation 3 is slightly difficult and time consuming. Here are some hints for you to do this task quickly. The key to doing this is "compare and linking".
  •  If you identify the same object in two different images, think about how you can use that object as a "pivot" to link two images.
  •  Find out whether two images share something in common, while there are still differences, so that you can ask a question to compare them.

Instructions
Desired Properties of a good question:
  • Not easy to be answered by pure text
  • No referential phrases directly referring to image regions
  • Has definitive answer
  • Not about simple facts
  • Associated with multiple image regions
  • Associated with more than one image (**required for Annotation 3, optional for Annotation 1&2**)
  • Non binary(**optional**)

Check more information by clicking "Show more image info". Feel free to look at the Wikipedia page or search online for the place/object/event related to that image. This is helpful for coming up with a complex question!
